🟡 OC-CFG-001: Gateway authentication not configured INFO WARN
No authentication mode set for the Gateway WebSocket server. Any local process or website (ClawJacked) can control the agent.
Recommendation: Set gateway.auth.mode to 'token' with a minimum 32-character random value.
🟡 OC-CFG-002: Gateway bound to all interfaces (0.0.0.0) INFO WARN
Gateway listens on all network interfaces, exposing the agent to the local network and potentially the internet.
Recommendation: Set gateway.bind to 'loopback' (127.0.0.1).
🟡 OC-CFG-003: mDNS/Bonjour broadcasting enabled INFO WARN
OpenClaw broadcasts its presence via mDNS, leaking filesystem paths and metadata to devices on the local network.
Recommendation: Set discovery.mdns.mode to 'off' or OPENCLAW_DISABLE_BONJOUR=1.
🟡 OC-CFG-004: Shell execution not restricted INFO WARN
tools.exec.security is not set to 'deny'. The agent may execute arbitrary shell commands, enabling prompt-injection-to-RCE.
Recommendation: Set tools.exec.security to 'deny' and tools.exec.ask to 'always'.
🟡 OC-CFG-005: Tool approval not required INFO WARN
tools.exec.ask is not 'always'. The agent can invoke tools without user confirmation.
Recommendation: Set tools.exec.ask to 'always'.
🟡 OC-CFG-006: Dangerous tool groups not denied INFO WARN
Control-plane tools (gateway, cron, sessions_spawn, sessions_send) or groups (automation, runtime, fs) are not explicitly denied.
Recommendation: Add gateway, cron, sessions_spawn, sessions_send, group:automation, group:runtime to tools.deny.
🟡 OC-CFG-007: Sandbox mode disabled or insufficient INFO WARN
Agent sandbox is not set to 'all'. Code may execute directly on the host without containment.
Recommendation: Set sandbox.mode to 'all' for full Docker isolation.
🟡 OC-CFG-008: Elevated tools enabled INFO WARN
tools.elevated.enabled is true. Elevated tools bypass sandbox restrictions and execute on the host.
Recommendation: Set tools.elevated.enabled to false unless specific elevated tools are required and individually approved.
🟡 OC-CFG-009: DM policy set to 'open' INFO WARN
Direct message policy allows anyone to interact with the agent without authentication or pairing.
Recommendation: Set dmPolicy to 'pairing' or 'allowlist'.
🟡 OC-CFG-010: Session DM scope not isolated INFO WARN
session.dmScope is not 'per-channel-peer', allowing context leakage between different message senders.
Recommendation: Set session.dmScope to 'per-channel-peer'.
🟡 OC-CFG-011: Browser SSRF protection disabled INFO WARN
browser.ssrfPolicy.dangerouslyAllowPrivateNetwork is true or browser mode allows unrestricted navigation.
Recommendation: Set dangerouslyAllowPrivateNetwork to false. Disable browser if not required, or use hostnameAllowlist.
🟡 OC-CFG-012: Workspace-only filesystem not enforced INFO WARN
tools.fs.workspaceOnly is not true. The agent has access to the entire user home directory.
Recommendation: Set tools.fs.workspaceOnly to true.
🟡 OC-CFG-013: Container namespace join allowed INFO WARN
dangerouslyAllowContainerNamespaceJoin is true, allowing sandbox escape via container namespace access.
Recommendation: Set dangerouslyAllowContainerNamespaceJoin to false.
🟡 OC-CFG-014: Auto-update not disabled INFO WARN
Automatic updates are not disabled. Enterprise deployments should use a controlled update pipeline where security evaluates each release.
Recommendation: Set updates.autoUpdate to false. Use controlled update pipelines.
🟡 OC-CFG-015: Moltbook heartbeat connectivity enabled INFO WARN
Moltbook connectivity is not explicitly disabled. The 4-hour heartbeat polling moltbook.com is a documented C2 vector; 1.5M tokens were exposed in the Moltbook database breach.
Recommendation: Set moltbook.enabled to false and moltbook.heartbeat.enabled to false.
🟡 OC-CFG-016: Gateway auth token too short INFO WARN
Gateway authentication token is shorter than 32 characters, making it vulnerable to brute-force attacks.
Recommendation: Use a minimum 32-character random token for gateway.auth.token.
🟡 OC-CFG-017: config.patch not restricted INFO WARN
config.patch is not in the tools deny list. OpenClaw config restrictions cover config.apply and config.update but omit config.patch, allowing unrestricted configuration modification.
Recommendation: Add config.patch to tools.deny list.
🟡 OC-CFG-018: Sensitive directories not excluded from agent access INFO WARN
File system exclusions do not cover critical sensitive directories (~/.ssh, ~/.aws, ~/.gnupg, ~/.kube, browser profiles, keychains, cryptocurrency wallets). A compromised agent can access these.
Recommendation: Add ~/.ssh, ~/.gnupg, ~/.aws, ~/.kube, ~/.docker, browser profiles, and keychain directories to tools.fs.exclude.
